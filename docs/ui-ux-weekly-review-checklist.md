# Weekly UI/UX Review Checklist

Use this checklist to review every route weekly. Score each route, record blockers, and assign owners.

## Core Metrics with Suggested Thresholds

1. Task success rate: >=85% for primary task on route.
2. Time to first action: <=10 seconds on workflow pages.
3. Completion rate: >=60% for core conversion flow steps.
4. Drop-off concentration: no single step contributes >35% of total drop-off.
5. Error rate: <5% of sessions hit form or navigation errors.
6. Scroll burden: avoid default states requiring full-page scans; use progressive disclosure.
7. Core Web Vitals:
   LCP <=2.5s
   INP <=200ms
   CLS <=0.1
8. Accessibility baseline: keyboard flow complete, visible focus, contrast pass, form labels connected.
9. Navigation clarity: users reach intended destination in <=2 clicks from common entry points.
10. CTA clarity: one primary and one secondary CTA above fold on conversion pages.
11. Trust signal presence: privacy/confidentiality/security cues visible near key actions.
12. Content scanability: heading ladder and section chunking supports 10-second skim.

## Top 12 First Principles

1. Clarity over cleverness.
2. User goal before company goal.
3. Minimize cognitive load.
4. Strong visual hierarchy.
5. Consistent interaction patterns.
6. Immediate, clear feedback.
7. Error prevention first.
8. Accessibility as default quality.
9. Performance is part of UX.
10. Content is interface.
11. Measure behavior, then opinion.
12. Iterate in short loops.

## Weekly Route Triage

1. High risk: score <80, owner assigned this week.
2. Flagged: score 80-89, owner assigned this sprint.
3. Excellent: score >=90, monitor for regression only.
