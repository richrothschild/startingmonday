# Luxury-Modern Redesign Brief: Homepage and Channel Entry Pages

Owner: Product + Design + Growth
Date: 2026-06-13
Scope: Homepage plus four primary channel entry pages.

## Goals

1. Raise perceived luxury and modern polish without losing clarity and urgency.
2. Keep usefulness explicit in the first viewport.
3. Increase trust and conversion intent through premium proof packaging.
4. Standardize page architecture across entry pages while preserving channel-specific messaging.

## Core Creative Direction

1. Visual tone: calm authority, premium restraint, editorial clarity.
2. Typography: high-contrast headline style plus highly legible body style.
3. Color system: neutral foundation, one signature accent, one confidence accent.
4. Layout rhythm: larger vertical spacing, fewer blocks, higher signal-to-noise ratio.
5. Motion: restrained reveal and state transitions only where they improve comprehension.

## Shared Component Changes Across All 5 Pages

1. Replace current dense hero with a luxury hero system.
2. Add PremiumProofStrip component directly below hero on each page.
3. Add EvidencePanel component with one quantitative outcome and one customer quote.
4. Add TrustRail component with privacy, security, and control guarantees.
5. Standardize CTA system.

CTA system copy:
- Primary CTA: Start free 30-day trial
- Secondary CTA: Review method and evidence
- Privacy microcopy: No credit card. No employer visibility. Privacy-first by design.

## Design QA Checklist for This Redesign

1. Every page has one primary CTA and one secondary CTA above the fold.
2. Every page has one quantified proof strip before scroll depth 35%.
3. Every page includes explicit privacy and trust language.
4. Every page includes a clear operating cadence statement.
5. Every page reduces copy density and improves visual spacing versus current baseline.
6. Every page can be scanned for value proposition in under 10 seconds.

## Implementation Notes

1. Reuse existing LandingPage where possible for homepage and executives channel.
2. Create reusable premium modules for trust, proof, and comparison blocks to avoid drift.
3. Instrument all new CTA and section interactions with existing channel metrics events.
4. Run council scoring and visual QA before launch.
