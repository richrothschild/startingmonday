# Sprint 1 Artifact: Founder Narrative Kit

Date: 2026-05-25
Owner: Founder Office
Status: Draft for Sprint 1 execution

## Narrative Standard

Each founder message should include:

1. one operating truth
2. one practical mechanism
3. one measurable signal
4. one direct call to action

## Story Block 1: The Problem Frame

Operating truth:

Most executive transitions fail early because momentum is unmanaged before search pressure peaks.

Use when:

1. top-of-funnel posts
2. opening talk tracks
3. first paragraph of outbound email

## Story Block 2: The Alternative Frame

Operating truth:

The real alternative is not another product. It is reactive, manual, inconsistent process.

Use when:

1. discovery calls
2. sales page sections
3. proposal framing

## Story Block 3: The Mechanism Frame

Operating truth:

EMI works through a daily momentum loop, weekly review cadence, and evidence-aware decision support.

Use when:

1. product walkthroughs
2. partner demos
3. onboarding explainer content

## Story Block 4: The Proof Frame

Operating truth:

Claims are only trusted when denominator, timeframe, and confidence are explicit.

Use when:

1. benchmark updates
2. thought leadership pieces
3. procurement and legal reviews

## Story Block 5: The Identity Frame

Operating truth:

Disciplined executives do not wait for disruption to start preparing. They run transition readiness as an operating system.

Use when:

1. keynote close
2. partner one-pagers
3. founder profile and about page

## Standard CTA Library

1. Run the EMI assessment and choose your operating mode.
2. Start the seven-day daily loop and measure real execution change.
3. For coaches and firms: run a 30-day pilot and compare session quality.

## Narrative Guardrails

Do:

1. use direct language
2. tie every claim to a mechanism or metric
3. keep one point of view per asset

Do not:

1. stack multiple frameworks in one short-form post
2. make unbounded claims
3. dilute language with generic productivity framing
