# Proof Asset Execution Package V1

Purpose: close the proof gap with repeatable assets that support premium positioning and partner trust.

Owner: Revenue lead
Review cadence: weekly

## Asset stack

1. Named executive case studies (minimum 3 live)
2. Coach testimonials (minimum 3 live)
3. Search-firm testimonials or pilot outcome quotes (minimum 2 live)
4. Sample output set (minimum 4 artifacts)
5. Methodology note for key claims

## Case study template (executive)

Use this format exactly.

- Profile: [title level], [industry], [company size], [search context]
- Starting challenge: [clear before-state in one sentence]
- What they used: [specific Starting Monday workflows used]
- Time horizon: [start date] to [milestone date]
- Measurable outcomes:
  - [metric 1]
  - [metric 2]
  - [metric 3]
- Executive quote: [approved quote]
- Approval status: [approved for name/logo or anonymized]
- Confidentiality level: [public / partial anonymization / full anonymization]

Acceptance rule:

- No case study is publishable without at least two measurable outcomes and one approved quote.

## Testimonial request templates

### Executive testimonial request

Subject: Quick request: your Starting Monday results

Body:

Hi [Name],

You used Starting Monday during a high-stakes transition, and your perspective would help other executives evaluate it responsibly.

Would you be open to sharing a short testimonial with:

- what problem you had before,
- what changed while using it,
- and one measurable outcome?

If easier, I can draft from your notes and you can edit.

Thank you,
[Sender]

### Coach testimonial request

Subject: Quick quote request on client session quality

Body:

Hi [Name],

We are documenting how coaches use Starting Monday as an execution layer between sessions. Would you be willing to share a short quote on:

- prep time impact,
- session quality impact,
- and client momentum impact?

One concrete example is enough.

Thank you,
[Sender]

### Search-firm testimonial request

Subject: Quick feedback request on readiness workflow

Body:

Hi [Name],

We are validating whether Starting Monday improves candidate-readiness conversations without disrupting retained-search workflow.

Would you share one short quote on what improved, what did not, and where trust boundaries mattered most?

Thank you,
[Sender]

## Sample output set (required)

Required artifacts:

1. Executive opportunity intelligence brief (sanitized)
2. Company signal scan (sanitized)
3. Coach session prep artifact (sanitized)
4. Search conversation readiness artifact (sanitized)

For each sample output include:

- Use case
- Input context
- Output excerpt
- Decision it enabled

## Methodology note template

Use this for any published performance claim (for example interview-rate claims).

- Claim statement: [exact wording]
- Time window: [dates]
- Population: [who is included/excluded]
- Definition details:
  - First interview rate definition
  - Offer rate definition
  - Time-to-milestone definition
- Data source(s): [product logs, CRM, user confirmation]
- Sample size: [n]
- Caveats: [material limits]
- Owner and approval date: [name/date]

Publication rule:

- No claim may be used in outreach, landing pages, or one-pagers without a linked methodology note.

## Weekly proof review checklist

1. Count live case studies by segment.
2. Count live partner testimonials by segment.
3. Confirm methodology notes for all numeric claims.
4. Confirm sample output set is current and sanitized.
5. Remove stale or unverified proof assets.
