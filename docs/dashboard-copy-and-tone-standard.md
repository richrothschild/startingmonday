# Dashboard Copy And Tone Standard

Status: Standard
Owner: Product + Content + Engineering
Last Updated: 2026-06-22
Primary Reference: [dashboard-modernization-plan.md](dashboard-modernization-plan.md)

## Purpose

Keep dashboard language calm, precise, and advisory. The page should feel like an executive brief, not a task app.

## Tone Requirements

Use:
- position
- timing
- next move
- follow-through
- keep momentum

Avoid unless strictly necessary:
- urgent
- due
- recovery
- accountability
- backlog

## Copy Structure Rules

Each recommendation must include:
1. What changed
2. Why it matters now
3. What to do next

Keep sections short and skimmable:
- Headings should be plain language
- Body lines should be concise and concrete
- Prefer one sentence to one paragraph when possible

## Label Standards

Preferred section labels:
- What matters now
- Signals to review
- People to reach
- Keep momentum
- Best next moves

Avoid labels that feel punitive or operationally noisy.

## CTA Standards

- One primary CTA per major section.
- CTA text should describe the next move, not generic navigation.
- Support CTAs should reduce effort, for example:
  - Why this matters now
  - Email me this plan
  - Save as today's note

## Empty State Standards

Empty states must:
- Confirm there is no urgent action required
- Offer one or two clear next options
- Avoid guilt framing

## Definition Of Done

Copy is complete when:
- Tone remains advisory and composed across all visible text.
- User can identify the first action quickly.
- No section introduces motivational hype or gamification language.
- Labels and CTA text pass this standard and [dashboard-page-architecture-standard.md](dashboard-page-architecture-standard.md).
